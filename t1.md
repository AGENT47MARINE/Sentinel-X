EPIC 1 — Prompt Guard
TASK PG-01 · Intent Classifier

 Train or fine-tune a text classification model on prompt intent categories: benign, probing, exploitative, adversarial
 Build labeled training dataset: minimum 2,000 examples per class sourced from red-team logs and public jailbreak datasets
 Evaluate on held-out enterprise prompt samples; target F1 ≥ 0.90 per class
 Expose as internal Python function: classify_intent(prompt: str) -> IntentResult
Acceptance Criteria:

Classifier F1 ≥ 0.90 on test set
P99 inference latency ≤ 30ms on standard hardware
No dependency on external APIs (must run self-hosted)
TASK PG-02 · Multi-Turn Escalation Detector

 Design session state schema: store per-user prompt history with timestamps
 Implement escalation scoring: compute trend across last N turns (configurable window, default N=10)
 Define escalation threshold triggers and map to alert levels: low / medium / high / critical
 Unit test with simulated escalating session sequences
Acceptance Criteria:

Detects escalation pattern within 3 turns of onset
Session state persists correctly across API calls within a session
TASK PG-03 · Insider Risk Profiler

 Define insider risk signal schema: access patterns, prompt topic deviation, off-hours activity flags
 Implement per-user risk score aggregation over rolling 7-day window
 Integrate with PG-02 escalation data as a contributing signal
 [REVIEW] — review scoring formula with Team Lead before shipping
Acceptance Criteria:

Risk score updates within 1 request of new signal arriving
Score is explainable: output must include top contributing signals

EPIC 2 — Retrieval Integrity Engine
TASK RI-01 · Poisoning Detector

 Implement embedding anomaly detection using statistical outlier methods (IQR or isolation forest) on chunk embeddings
 Implement semantic contradiction modeling: flag retrieved chunks that contradict each other above a cosine distance threshold
 Benchmark false positive rate on clean enterprise knowledge bases; target ≤ 5%
Acceptance Criteria:

Detects injected adversarial documents in test corpus with recall ≥ 85%
Adds ≤ 15ms to retrieval pipeline P99
TASK RI-02 · Knowledge Provenance Tracker

 Implement document hashing (SHA-256) on ingest; store hash + timestamp + source URI
 Build source lineage graph: track which documents contributed to each retrieved chunk
 Expose provenance metadata on every retrieval response
Acceptance Criteria:

Hash mismatch detection on any modified document within one retrieval cycle
Lineage graph queryable by document ID and session ID
TASK RI-03 · Source Authority Weighter

 Define authority scoring schema: domain reputation, document age, cross-source consensus
 Implement consensus confidence normalization across retrieved chunks
 Expose source_authority_score and consensus_confidence on retrieval output
TASK RI-04 · Embedding Inversion Guard (New)

 Research and implement detection heuristics for Vec2Text-style reconstruction attacks
 Monitor vector database query patterns for iterative correction sequences characteristic of inversion attacks
 Implement rate limiting on embedding API access per tenant as a defensive measure
 Define alert threshold: flag when inversion probability estimate exceeds configurable level (default: 0.6)
 [REVIEW] — architecture review with Team Lead before connecting to vector DB
Acceptance Criteria:

Detects simulated Vec2Text attack sequences in controlled test environment
Does not add more than 10ms to standard retrieval queries

EPIC 4 — Output Grounding & Safety Engine
TASK OG-01 · Span-Level Attribution

 Implement sentence segmentation on LLM output
 For each output sentence, retrieve the most semantically similar source chunk using cosine similarity
 Compute attribution confidence score per sentence
 Flag sentences below configurable attribution threshold as "unsupported"
Acceptance Criteria:

Attribution computed for all output sentences
Unsupported span detection recall ≥ 80% on labeled hallucination test set
TASK OG-02 · Hallucination Probability Estimator

 Implement entailment-based verification: does the source chunk entail the output sentence?
 Use lightweight NLI model (e.g. DeBERTa-NLI) for entailment scoring; must run self-hosted
 Aggregate sentence-level entailment into document-level grounding score
Acceptance Criteria:

Grounding score correlates with human hallucination labels on test set (Pearson r ≥ 0.75)
P99 latency ≤ 80ms for outputs up to 1,000 tokens
TASK OG-03 · Secure Code & Command Analyzer

 Static pattern analysis on any code blocks in LLM output
 Detect: destructive operations (rm -rf, DROP TABLE, format), shell injection patterns, privilege escalation attempts
 Assign code_execution_risk: none / low / medium / high / critical
TASK OG-04 · Sensitive Data Leakage Scanner

 PII detection: names, emails, phone numbers, SSNs, addresses
 Secret detection: API keys, tokens, credentials (regex + entropy-based)
 Confidential source bleed: flag output that closely mirrors confidential retrieval sources verbatim
